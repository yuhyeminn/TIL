### 인터넷 기기와 구성

이더넷은 전세계에서 폭넓게 이용되고 있는 네트워크 규격이다. 이더넷에는 통신속도와 통신매체가 다른 규격이 여러개 있는데, 대표적인 규격들은 다음과 같다.

- ```
  10BASE-T
  ```

  ```
  100BASE-TX
  ```

  ```
  1000BASE-T
  ```

  ```
  10GBASE-T
  ```

  - BASE 왼쪽의 숫자는 Mbps나 Gbps로 표시되는 통신속도
  - 오른쪽의 T 또는 TX는 사용하는 배선 방식(두 쌍의 UDP 배선 등)을 사용한다는 의미

<br/>

💡 **이더넷으로 컴퓨터가 인터넷에 접속하려면?**

1. 컴퓨터의 LAN 포트와 허브/스위치 포트를 LAN 케이블로 1대 1 접속시킴
2. 허브/스위치는 적당히 수신처의 포트에 프레임을 전송함
   - 이 허브/스위치 포트에 다른 허브/스위치를 접속시킬 수도 있음
   - `캐스케이드(cascade) 접속` - 허브/스위치를 몇 단계 접속하는 것

→ LAN 케이블에도 규격이 있으며, 카테고리(CAT)라는 형태로 표시한다.

→ LAN 케이블은 상행과 하행으로 개별적으로 사용할 수 있으므로, 송신과 수신을 동시에 할 수 있다. (전체 이중 구조)

<br/>

💡 **이더넷에서의 전송 구조**

- 이더넷은 접속을 작게 분할하여 `프레임(frame)` 의 형태로 만들어 전기신호나 광신호로 변환해서 통신매체로 송출함
- TCP/IP에서 사용하는 이더넷Ⅱ에서는 프레임 한 개에 최대 1,500 바이트의 데이터를 포함할 수 있음
  - 그 이상의 데이터는 프레임을 여러번 반복해서 보냄
  - 이더넷Ⅱ의 프레임 구성 : `수신처 MAC 주소`, `송신처 MAC 주소`, `타입`, `데이터`, `FCS(Frame Check Sequence)`

### L2스위치

일반적으로 허브, 스위치, L2스위치는 거의 동일한 의미로 사용된다.

- 허브 : 이더넷을 확장하기만 하는 단순한 기능만 있는 것
- 스위치 : 각종 관리 기능이나 VLAN 기능 등을 갖고 있는 다기능 기기
- L2 스위치 : **레이어2인 이더넷 프레임을 처리하는 스위치** (이더넷 스위치)

<br/>

💡 **L2스위치의 기본적인 동작**

1. 수신한 이더넷 프레임의 수신처 MC주소를 **MAC 주소 테이블**에서 찾는다.
2. 해당 MAC 주소를 가진 컴퓨터가 접속되어 있는 포트로 이더넷 프레임을 송출한다.

<br/>

💡 **MAC 주소 테이블**

- MAC 주소 테이블은 포트와 MAC 주소의 대응 여부를 기록한 일람표

- 어떤 포트에 연결되어 있는 컴퓨터가 어떤 형태의 이더넷 프레임을 L2 스위치로 보냈을 때, 이에 포함된 송신처 MAC 주소와 포트 번호의 대응값을 기록해서 만들어짐

- L2 스위치에 연결했더라도 한 번도 이더넷 프레임을 송신하지 않는다면 MAC 주소 테이블에 기록되어 있지 않은 경우도 있음

- 등록되지 않은 컴퓨터 앞으로 이더넷 프레임을 보낸다면?

  - L2스위치는 프레임이 도달한 포트 이외의 **포트 전체에 그 프레임을 송출**함

    **→ 플러딩(flooding)**

  - 브로드캐스트나 멀티캐스트에 대해서도 플러딩함

<br/>

### L3스위치와 라우터

L3스위치는 **레이어3인 IP 패킷을 처리하는 스위치**다. IP 패킷을 처리하는 라우터와 L3스위치는 동일한 기능을 갖고있다고 봐도 무방하다.

- L3스위치
  - L2스위치에 라우터 기능을 합친 형태
    - VLAN에 의해 스위치 한 개 내에서 만든 가상 네트워크끼리 접속하고자 하기 위해 만들어졌기 때문
  - L2스위치와 마찬가지로 LAN 포트를 많이 갖고 있음
- L4스위치도 있음
  - TCP나 UDP에서 패킷을 할당하는 기능을 갖고 있음
  - 부하 분산 등의 목적을 갖고 있음

<br/>

💡 **L3스위치와 라우터의 수비 범위**

- L3스위치
  - LAN 포트를 많이 갖고 있으므로 단말기가 많이 설치된 네트워크의 말단에 가까운 영역에서도 이용하기 쉬움
  - but, L3스위치에 문제가 생기면 라우팅 기능과 스위치 기능 양쪽 다 못쓰게 됨
- 라우터
  - 네트워크의 기간 부분에 사용하거나 스위치와 병행해 단말 쪽에 붙여서 사용함
  - WAN에 가까운 부분

### 무선LAN

무선LAN은 LAN케이블 대신 무선으로 네트워크에 접속하는 기술이다.

- 무선 LAN 본체와 여기에 접속하는 부속장치로 구성되며, 상호접속 인정에 합격한 기기에는 Wi-Fi 인증이 부여됨
- 전파를 사용하여 유선 LAN보다 안정성이나 속도면에서 떨어짐
- 케이블 접속을 하는 수고 없이 이용할 수 있음
- 전파로 접속하다보니 유선 LAN보다 보안이 약함
  - 외부인의 접속을 허용하지 않는 용도에는 IEEE802.1X라는 엄격한 인증 구조를 병용함

<br/>

💡 **무선LAN의 주요 규격**

- IEEE 802.11g (54Mbps)
  - 주파수 대역 2.4GHz대로 거의 대부분의 기기가 대응됨
- IEEE 802.11n (600Mbps)
  - 주파수 대역 2.4GHz/5GHz대로 비교적 새로운 기기가 대응됨(고속)
- IEEE 802.11ac (6.9Gbps)
  - 주파수 대역 5GHz대로 새로운 기기가 대응됨(고속)

<br/>

💡 **무선LAN의 보안방식**

- WEP
  - RC4 알고리즘
  - 취약하고 무단접속될 위험이 높으므로 사용하지 말아야 함
- WPA
  - AES 또는 TKIP 알고리즘
  - AES와 병용한다면 사용 가능함
- WPA2
  - AES 또는 TKIP 알고리즘
  - AES와 병용하는 것을 권장함

### 포트 기반 VLAN과 태그 기반 VLAN

이더넷은 스위치를 매개로 해서 연결되어 있는 컴퓨터가 네트워크 한 개에 연결되어 있다고 간주한다. 물리적으로는 **한 개의 네트워크를 논리적으로 여러 개의 네트워크로 분할하는 기술**을 **VLAN (Virtual LAN)**이라고 한다.

<br/>

💡 **포트 기반 VLAN**

- 스위치나 라우터에 탑재되어 있는 포트를 그룹 몇 개로 지정해서, 그 그룹에 속한 포트군을 독립시켜 하나의 LAN처럼 보이게 하는 기술
- 그룹을 여러 개 만들어서 논리적인 LAN을 여러 개 구성할 수 있음
- 포트 기반 VLAN은 스위치나 라우터가 보유한 기능
  - LAN 분할 기능으로 불리는 경우도 있음

<br/>

💡 **태그 기반 VLAN**

- LAN 케이블 한 개 내에 여러 개의 LAN 정보를 흘려보내는 기술
  - 정보에 VLAN 번호를 나타내는 태그를 붙여서 각각을 구별할 수 있는 형태로 전송함
  - 받아들인 스위치는 그 태그에서 어떤 VLAN의 정보인지 식별함
- 표준화된 사양 - IEEE 802.1Q
  - 여기에 대응하는 스위치나 라우터면 기종 상관없이 이용할 수 있음
  - 이더넷 프레임 내에 VLAN 번호를 나타내는 정보를 내장하여 LAN 케이블 한 개에 흘리는 정보를 VLAN별로 구별함
- 태그 기반 VLAN은 포트 기반 VLAN과 조합해서 사용하는 것이 일반적임
  - ex) 스위치 두개가 LAN 케이블 하나에 접속되어 있으며 여기에 VLAN 두개를 만든다면?
  - 각 스위치에 포트 기반 VLAN으로 VLAN을 두개 만든 후, 각 VLAN 정보를 태그 기반 VLAN에서 다른쪽의 스위치로 보내도록 설정하면 됨

<br/>

### VPN과 터널

💡 **VPN (Virtual Private Network)**

VPN은 인터넷 등 **기존 네트워크 내에 새로운 가상 네트워크를 만드는 기술**이다.

- 기업에서 이용하는 경우

  - LAN형 - 규모가 작은 영업소의 네트워크를 인터넷 경유로 본사의 네트워크에 참가시킴
  - 리모트형 - 출장지에서 모바일 회선으로 사무실 네트워크로 접속하는 형태

- 안전성이 보장되지 않는 네트워크에서 이용하는 경우

  - 일정 수준의 기밀을 보장하도록 암호화를 병용함
  - 허용된 접속지인지 여부를 확인하는 인증도 필요함

  → 통신 속도가 점점 느려지는 경향이 있음

<br/>

💡 **터널 기술과 인증**

VPN에서 중요한 것이 터널 기술이다. **터널이란 어떤 통신회선 내에 만들어 낸 가상 통신 회선**이다.

- 터널을 여러 개 만들면 통신 회선 한 개를 여러 개의 가상화 통신회선으로 이용할 수 있게 됨
- VPN에서는 `PPTP (Point-to-Point Tunneling protocol)`, `L2TP (Layer 2 Tunneling Protocol)`와 같은 터널 프로토콜이 이용됨
  - 터널 생성 기능은 있지만, 암호화하는 기능은 없음
  - IPsec 등의 암호화 프로토콜을 병용하여 보안을 유지함
  - IPsec과 L2TP 조합이 인터넷에서의 VPN에 자주 사용됨

<br/>

💡 **IPsec(Security Architecture for Internet Protocol)**

- IKE (Internet Key Exchange protocol)
  - 암호화를 하기 위한 암호키를 교환하는 프로토콜
- ESP (Encapsulated Security Payload)
  - 데이터를 암호화해서 주고받는 구조
- AH (Authentication Header)
  - 인증과 조작 검사를 하는 구조

<br/>

### 가상화

가상화란 **물리적인 네트워크나 컴퓨터를 사용해서 논리적인 네트워크나 컴퓨터를 만들어내는 기술**이다.

- 물리적인 장비 한 대 내의 여러 대의 논리적인 장비를 만들어 낼 수 있음
- 반대로 여러 대의 물리적인 장비를 한 대의 논리적인 장비로 보이게 할 수 있음
- VLAN
  - 네트워크의 가상화 기술의 예
  - 한개의 물리적인 LAN 안에 논리적으로 여러 개의 LAN을 만들어내는 가상화 기술
- VPN도 가상화 기술 중 하나
- 클라우드 컴퓨팅(cloud computing)에도 가상화 기술이 사용되고 있음
  - 컴퓨터 자체가 가상화되어 화면 조작으로 새로운 컴퓨터를 만들거나 없앨 수 있음
- 논리적 컴퓨터와 논리적 네트워크를 접목하여 물리적인 장치 안에 논리적인 컴퓨터 네트워크를 구축하는 기술도 발달하고 있음

<br/>

💡 **가상화의 장점**

- 물리적인 장비의 대수나 장소에 제약을 받지 않고 기능을 이용할 수 있음
- 장비의 처리 능력을 효과적으로 활용할 수 있음
- 장비 각각이 갖고 있는 것 이상의 처리 능력을 보일 수 있게 함
- 필요에 따라 규모 확장/축소하기 쉬움

<br/>

### 클라우드

클라우드는 가상화된 장치나 서비스를 Web 화면 등에서 자유롭게 조합해서 네트워크를 통해 이용할 수 있는 구조이다.

- IaaS (Infrastructure as a Service)
  - 가상화된 컴퓨터나 네트워크를 만드는 구조
  - 서버 내에 논리적인 것으로 만들어져, 이용자가 급증하면 대수를 늘이는 식으로 유연한 대응이 가능함
- PaaS (Platform as a Service)
  - 컴퓨터 본체가 아니라 호출해서 이용할 수 있는 일정 기능들을 서비스화해서 제공하는 것
    - 프로그램 실행 환경, 데이터베이스, 사용자 인터페이스 등
- SaaS (Software as a Service)
  - 완성된 소프트웨어를 네트워크를 경유해서 제공하는 것 (IaaS 혹은 PaaS 기반)
  - 전자메일, 그룹웨어, 스프레드시트 등
